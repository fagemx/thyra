# Thyra 真需求測試框架 v1

## 先記一句總原則
**不要問：這個世界酷不酷。**

# 一、7 個測試問題

每題 **0-2 分**：

- **0 分** = 幾乎沒有
- **1 分** = 有一點，但很弱
- **2 分** = 很明確，很強

總分 **14 分**。

---

## 1. Place Test — 它真的是「一個地方」嗎？
**問題：**
人類會不會把它理解成一個「地方」，不是一個功能，不是一堆資訊，不是一個 bot？

### 看什麼
- 有沒有明確場景感
- 有沒有進入感、停留感、發生事情的感覺
- 人是不是會說「我去那裡看看」

### 例子
- Midnight Market：**2**
- AI 寵物百科站：**0**
- Creator Freeport：**2**
- 農場內容網站：**0**

---

## 2. Ops Density Test — 每天真的有事情要營運嗎？
**問題：**
如果沒有人碰它，這個地方每天有沒有很多事 still 必須被決定？

### 看什麼
- 活動排程
- 價格/資源分配
- 人流秩序
- 新角色 / 新事件
- 衝突處理
- 成長與穩定的取捨

### 關鍵
**沒有高營運密度，就不是 Thyra。**

### 例子
- 夜市 / 港口 / live event 場：**2**
- 純虛擬度假島：**0-1**
- AI 菜園如果只是展示：**0**
- AI 菜園如果有地塊、季節、訂閱盒、採收日：**2**

---

## 3. Conflict Test — 有沒有真的會互相打架的目標？
**問題：**
裡面有沒有天然衝突？沒有衝突就沒有治理，沒有治理就不需要 Thyra。

### 典型衝突
- 收入 vs 體驗
- 熱門攤位 vs 公平
- 新手友善 vs 高價值玩家
- 世界觀一致性 vs 商業化
- 開放 vs 安全
- 成長 vs 穩定

### 判準
如果只是在做推薦、整理、導覽，通常是 **0 分**。

---

## 4. Persistence Test — 人不在時，它還會自己變嗎？
**問題：**
當你離開 8 小時 / 24 小時後，回來會不會真的看到：

- 新決策
- 新事件
- 新狀態
- 新後果
- 可追溯的變化

### 核心
**Thyra 不是回答器，是持續狀態機。**

如果「你不碰它就什麼都不發生」，那通常不是 Thyra 世界。

---

## 5. Human Return Test — 人為什麼會回來？
**問題：**
除了「很酷」之外，人第二次、第三次為什麼要回來？

### 可能答案
- 今晚活動不同
- 商品 / 角色 / 劇情變了
- 有交易 / 獎勵 / 會員機制
- 有社群互動
- 有持續推進的場域狀態

### 危險訊號
如果答案只有：
- 看看 AI 今天又生成了什麼
- 好玩一次就好
那多半是 **概念玩具**，不是需求。

---

## 6. Buyer Test — 誰會真的想為它付錢？
**問題：**
不是誰會說「好酷」，而是誰會說：

> 「如果這東西成立，我願意為它付錢 / 導入 / 維持。」

### 真正可能買單的人
- 創作者社群經營者
- 品牌方
- 活動方
- IP 持有者
- 遊戲/live ops 團隊
- 社群主理人

### 如果回答不出 buyer
那多半還在概念層。

---

## 7. Ugly Workaround Test — 現在人們有在用笨方法解這問題嗎？
**問題：**
如果需求是真的，通常人們早就在用醜陋但有效的方式硬解了。

### 要找什麼
- 人工排活動
- 手動調社群節奏
- 小編維持熱度
- admin 人工維護秩序
- 手工管會員 / 票券 / 內容 cadence
- 多個 bot + 表單 + Notion + Discord 拼起來

### 核心
**沒有 workaround，不代表沒需求；但有 workaround 是強訊號。**

# 1. Midnight Market

## 總分：14 / 14
幾乎是滿分模板。

### 1) Place Test — 2
這是一個非常明確的地方。

- 人一聽就知道是什麼
- 有逛、停留、路線、攤位、活動、熱點
- 天然不是工具，不是資訊站

它不是 abstract world，  
它是 **今晚要不要去逛的地方**。

### 2) Ops Density Test — 2
營運密度最高。

每天都可能要決定：
- 今晚開什麼活動
- 哪些攤位上熱門位
- 哪些商品供給太多 / 太少
- 人流怎麼導
- 價格怎麼調
- 哪些規則要回滾
- 怎麼防洗單、刷活動、破壞體驗

這種地方，AI 有非常多「住進去」的理由。

### 3) Conflict Test — 2
天然衝突超多：

- 收入 vs 體驗
- 熱門攤位 vs 公平
- 活動熱度 vs 安全
- 新手友善 vs 高消費客
- 商業化 vs 世界感
- 節奏感 vs 穩定性

這些衝突就是 Thyra 的養分。

### 4) Persistence Test — 2
你離開一晚，隔天一定有東西可看：

- 昨晚哪個活動爆了
- 哪個攤位被調位
- 哪個獎勵被 rollback
- 哪個規則留下 precedent

**「昨晚這裡發生了什麼」** 這句在 Midnight Market 最自然。

### 5) Human Return Test — 2
人會回來的理由很多：

- 今晚活動不同
- 攤位不同
- 商品不同
- 熱點不同
- 世界狀態不同

不是一次性 novelty。

### 6) Buyer Test — 2
最容易找到 buyer：

- 社群主
- 品牌活動方
- 創作者經營者
- 虛擬活動策展人
- 有商品 / 票券 / 會員需求的人

也最容易連到現實世界：
- 節慶
- 社群活動
- 攤位合作
- 品牌聯名
- 票券

### 7) Ugly Workaround Test — 2
現實裡已經有很多醜 workaround：

- Discord 活動 + Google 表單
- 小編人工炒熱度
- 人工排攤位 / 排活動
- Notion + bot + 社群 admin 拼起來
- 人工管規則和秩序

這很重要，代表它不是幻想中的問題。

## Midnight Market 的缺點
也不是沒缺點。

### 缺點 1：可能太強，強到把 Thyra 綁成 market-only
大家會以為：
> Thyra = AI 夜市系統

這是第一個風險。

### 缺點 2：容易被誤解成 live commerce / 活動策展工具

- 活動編排工具
- AI 商店系統
- 更 fancy 的 community ops

所以 demo 必須讓人看到：
- chief
- law
- rollback
- precedent
- continuity

不然會掉成「聰明營運 SaaS」。

- 交易

- 低摩擦 vs 信任
- 港口繁榮 vs 邊界穩定

這種衝突比 Midnight Market 更「制度級」。

這些 buyer 比 Midnight Market 更抽象，sales 也更難。

### 7) Ugly Workaround Test — 2
這種需求的 workaround 其實很多：

- 多平台社群切換
- 手動驗證與通行機制
- Discord role + whitelist + ticket system
- 人工審核、人工身份管理、人工交易規則

代表問題是存在的。
